Implementation

Every language in your system does one or more of:

  • Persist to database / filesystem
  • Validate input
  • De/serialise
  • Test—incl. mock—input and output
  • Document interfaces

Different languages—and frameworks—have different strength type systems, but in general one can:

  1. Traverse the Abstract Syntax Tree (AST)
  2. Extract pertinent information
  3. Generate OpenAPI JSON/YAML

Inversely, one can:

  1. Traverse the OpenAPI hierarchy
  2. Extract pertinent information
  3. Parse existing source and merge new information (if exists) else generate; to/from the AST

Type system incompatibilities

SQL has really specific types, like VARCHAR(20); almost no other language gets as specific.

Sure, you could use ADTs (implemented using your record syntax), but this usually incurs a runtime overhead.

To avoid the tradeoff between runtime complexity and type information loss, additional type information—beyond what the target language/framework supports—is added to the docstring.
